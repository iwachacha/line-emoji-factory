# アイデア探索・市場調査ルール

このファイルは、ブランド探索や絵文字アイデア出しで使う
市場調査、多様性確保、偏り抑制の判断基準を定義する。

## 境界
- 市場調査は `差別化` と `日常接続性` を補強するために使う。
- 市場で見つけた人気表現を、そのまま模倣しない。
- 市場観測は探索の入口であり、最終判定は常に `構造 → ブランド → 商品` で行う。
- 構造で `Fail` の案は、雰囲気や流行だけで残さない。

## 調査で見ること
- 今よく見かけるテーマ、温度感、構図、反応語彙
- 混みすぎている表現と、まだ薄い表現
- 高頻度会話に接続しやすい用途
- 季節、文化、生活イベントなどの時間要因
- 既存表現が強いが、使いにくさや見分けにくさが残る領域

## 多様性の回転軸
探索 batch では、必要に応じて次の軸を回す。
- テーマ / ジャンル
- `構造分類`
- ブランド主型
- 関係性の温度感
- 実用品寄りか感情寄りか
- 装飾密度
- 安定枠か尖り枠か

## 偏り抑制
- 直近の探索が同じ主題や温度感に寄っているなら、次 batch では意図的に外した候補を混ぜる。
- かわいさ、世界観、エモさのどれか 1 つだけで押し切らない。
- 毎回同じ構造分類やブランド主型に寄るなら、別軸の候補を必ず入れる。
- 尖った案は残してよいが、`標準枠` と `挑戦枠` の両方を比較できる状態を保つ。

## 出力原則
- 市場観測は `観測日 / 何を見たか / どう解釈したか / 真似しない線引き` を残す。
- 1 batch の案出しは、広げるだけで終わらせず構造の予備判定まで行う。
- 構造で危ういが核はある案は `workflows/transformation-workflow.md` へ送る。
- 強い市場 signal が見えても、ブランドとして痩せる案は通さない。

## 継続改善への接続
- どの軸に寄りやすかったかは、次 batch の偏り抑制材料として残す。
- 再発する探索の癖は、`rules/continuous-improvement-rules.md` に従って skill や workflow の改善候補に上げる。
