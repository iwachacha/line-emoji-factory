# 評価モデル

このファイルは、全レイヤーで使う評価語彙と判定状態の正本である。
他ファイルは、このファイルの語だけを使って判断する。

## 判定順序
1. ハード制約を確認する。
2. コア評価で強みと弱みを測る。
3. 調整可能要素で最適化余地を見る。
4. `Fail / Revise / Keep / Design Ready` のいずれかに落とす。

## ハード制約
- `公式仕様不一致`
  - `rules/line-platform-baseline.md` の画像仕様、パッケージ数、ファイル形式、アニメーション制約に反している。
- `構造不成立`
  - `rules/structure-constraints.md` を通らない。
- `視認性崩壊`
  - 小表示で主要要素が判別不能。
- `実装前提の誤認`
  - 任意文字列装飾や自動埋め込みなど、成立しない実装前提に依存している。
- `Hard NG リスク`
  - `rules/review-risk-rules.md` の `Hard NG` に当たる。

## コア評価
### ブランド寄り
- `核の明確性`
  - 一言で何者か説明できるか。
- `記号性`
  - 小さくなっても、そのブランドらしさが識別できるか。
- `圧縮性`
  - 情報量を減らしても成立するか。
- `差別化`
  - 他案や既存表現と混ざらないか。
- `日常接続性`
  - 会話の中で自然に使う場面が想定できるか。
- `派生性`
  - シリーズや派生ブランドへ展開できるか。
- `AI再現性`
  - `GPT / image_gen` の rough、anchor、item finalization、revision で、工程を跨いでも指示と結果が安定するか。
  - 1アイテム単位の候補比較と差し戻しが可能か。

### 商品寄り
- `視認性`
  - 小表示で主要要素が潰れないか。
- `即読性`
  - 見た瞬間に用途や反応が読めるか。
- `両用性`
  - 単体送信でも文中使用でも使えるか。
- `低摩擦性`
  - 相手や場面を選びすぎず添えやすいか。
- `再使用性`
  - 一度きりではなく繰り返し使えるか。
- `差分明確性`
  - セット内の差分が明確か。
- `セット整合性`
  - セット全体で核と視覚記号が揃っているか。
- `連続使用適性`
  - 2個3個並べてもノイズになりすぎないか。
- `量産性`
  - 新作や差分を安定して増やせるか。
- `商品転換力`
  - ブランドの核が実際の絵文字 / スタンプ商品へ自然変換されているか。

## 調整可能要素
- `温度感`
- `尖り具合`
- `世界観の濃さ`
- `対象層の狭さ`
- `表情の強さ`
- `情緒の重さ`
- `装飾密度`

## 判定状態
- `Fail`
  - ハード制約違反がある。
  - 先に変換か切り判断を行う。
- `Revise`
  - 構造は通るが、コア評価の弱点が大きい。
  - 修正方針を明示して再評価する。
- `Keep`
  - 明確な評価継続価値がある。
  - 比較候補または採用候補として残す。
- `Design Ready`
  - 設計モードへ進めるだけの条件が揃っている。
  - `templates/brand/brand-setting-template.md` を埋めてよい。

## 判定プロトコル
- `Hard NG` は即 `Fail`。
- `Revise` リスクは、原則 `Revise` 以上に引き上げない。
- `Watch` リスクは、ブランド設定、ブランド固有制作基盤、release 設計、制作ハンドオフ、品質台帳、専用AI制作指示へ引き継ぐ。
- `Design Ready` は、構造・ブランド・商品の3層で大きな破綻がない場合だけ使う。
