# Quickstart

## 1. Install Dependencies

```powershell
python -m pip install -r requirements-dev.txt
```

## 2. Create a Brand Repo

```powershell
./tools/init-brand-repo.ps1 `
  -BrandSlug "my-brand" `
  -BrandName "マイブランド" `
  -Destination ".\brands\my-brand" `
  -InitialSetCount 8
```

Use `-InitialSetCount` with one of `8`, `16`, `24`, `32`, or `40`.
The generated manifest also contains `production_profile`, which keeps rough, finalization, and revision outputs explicit without making a specific tool the source of truth.

For fixed IP projects, add:

```powershell
-BrandType fixed_ip
```

The manifest will require the IP governance files under `brand/ip/`.

## 3. Validate the Scaffold

```powershell
./tools/validate-brand-repo.ps1 ".\brands\my-brand"
```

This checks the manifest schema, referenced paths, required release directories, metadata, and unresolved placeholders.

## 4. Edit the First Files

- `brand/brand-setting.md`
- `brand/brand-positioning.md`
- `brand/brand-production-brief.md`
- `releases/release-001/release-spec.md`
- `releases/release-001/production-handoff.md`
- `releases/release-001/submission/metadata.yaml`

## 5. Add Images

Place the tab image here:

```text
brands/my-brand/releases/release-001/production/tab/source-tab.png
```

Place final content images here:

```text
brands/my-brand/releases/release-001/production/finals/*.png
```

Production filenames may be arbitrary. Submission filenames are generated by `package-release.py`.

## 6. Package

```powershell
python ./tools/package-release.py ".\brands\my-brand" `
  --release-id release-001 `
  --target both `
  --clean
```

Outputs:

- `submission/line-upload/images.zip`: LINE upload ZIP. Images only.
- `submission/internal-archive/package.zip`: internal archive with metadata, package report, asset map, checksums, manifest snapshot, and release spec snapshot.

## 7. Validate Again

```powershell
./tools/validate-brand-repo.ps1 ".\brands\my-brand"
python ./tools/validate-assets.py ".\brands\my-brand\releases\release-001\submission\line-upload\images" `
  --expected-count 8 `
  --stage submission `
  --preview-contact-sheet ".\brands\my-brand\releases\release-001\qa\contact-sheet.png"
```

Common failures:

- Missing `production/tab/source-tab.png`
- Fully transparent image
- Content image not `180 x 180`
- Tab image not `96 x 74`
- Submission filename not `001.png`, `002.png`, ...
- Metadata title, description, creator name, or copyright exceeds LINE limits
- Copyright contains non-ASCII characters

## Animation Assets

Validate APNG content images directly:

```powershell
python ./tools/validate-assets.py ".\brands\my-brand\releases\release-001\production\finals" `
  --expected-count 8 `
  --asset-type animation `
  --tab-image ".\brands\my-brand\releases\release-001\production\tab\source-tab.png"
```

Animation APNG validation checks frame count, duration, loop count, dimensions, transparency, and file size. `package-release.py` still packages static emoji only.

## Example Drift Check

`examples/soeshirushi` is a standalone generated repo sample. Check that its embedded schemas, tools, and shared snapshots match the factory sources:

```powershell
python ./tools/check-example-drift.py ".\examples\soeshirushi"
```
