# Startup Checklist

## Structure

- [ ] The brand can become at least one LINE item type.
- [ ] The first item type hypothesis is recorded.
- [ ] One image can carry a usable reaction, phrase, or meaning.
- [ ] The idea is not layout-dependent only.
- [ ] Transformation notes exist if structure is weak.

## Brand

- [ ] One-sentence definition exists.
- [ ] Brand main type and supporting tags are recorded.
- [ ] Hard canon is narrow.
- [ ] Variable elements are explicit.
- [ ] Similar IP / rights risks are excluded.

## Product

- [ ] First product hypothesis exists.
- [ ] First set count hypothesis has a reason.
- [ ] High-frequency items are listed.
- [ ] Support and signature items are listed.
- [ ] Next-series room is preserved.

## Production Readiness

- [ ] Prompt library exists.
- [ ] Negative prompt exists.
- [ ] Character / motif data exists.
- [ ] Item seed data exists.
- [ ] Watch items are listed.
- [ ] The repo is not carrying unnecessary submission or release artifacts.
