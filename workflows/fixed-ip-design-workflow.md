# 固定IP設計ワークフロー

このファイルは、ブランド創出後にキャラクター、モチーフ、記号体系を
固定IPとして運用できる状態へ落とすための標準手順を定義する。

固定IPは optional module である。
コンセプト、用途、視覚文法だけで成立するブランドでは、この workflow を必須にしない。
その場合は `brand-setting` と `brand-production-brief` に、固定する視覚文法と禁止 drift だけを残す。

固定IPは、見た目を固めるだけではなく、
権利の所在、再現できる造形文法、禁止 drift、商品展開時の変えてよい範囲を持つ。
判断順序は常に `構造 → ブランド → 商品` とする。

## 起動条件
- `Design Ready` 相当のブランドを、継続制作できる固定IPへ育てる。
- キャラクター、モチーフ、記号、図形・抽象、デコ文字の見た目を量産時に固定したい。
- GPT-image2.0 rough と ClaudeDesign 仕上げを跨いでも、同じIPとして見える必要がある。
- 絵文字以外の商品化や派生シリーズの可能性がある。

## 使う正本
- ブランド設定: `templates/brand-setting-template.md`
- 固定IPバイブル: `templates/fixed-ip-bible-template.md`
- ブランド制作基盤: `templates/brand-production-brief-template.md`
- 専用AI制作指示: `templates/brand-system-prompt-template.md`
- 審査・権利リスク: `rules/review-risk-rules.md`
- 品質管理: `workflows/quality-control-workflow.md`

## 標準手順
1. `rules/structure-constraints.md` で、IPの表現単位が1画像単位でも成立するか確認する。
2. `rules/brand-creation-rules.md` で、核、視覚記号、派生性、AI再現性を確認する。
3. 権利の所在を確認する。
   - 完全オリジナルか。
   - 参考元、素材、既存IP、写真、著名人、商標、ロゴに依存していないか。
   - 許諾が必要な要素がないか。
4. IPの固定要素を決める。
   - シルエット
   - 顔 / 目 / 口 / 手足 / パーツ比率
   - 線幅、角、塗り、色、余白
   - 口調、温度感、反応語彙
   - キャラクター同士の関係性
5. IPの可変要素を決める。
   - 表情、ポーズ、持ち物、季節小物、色差分、演出記号。
   - ただし小表示で識別できる範囲に限る。
6. 禁止 drift を決める。
   - 別キャラクター化、既存IP寄せ、ロゴ化、広告記号化、過剰装飾、細線化など。
7. `templates/fixed-ip-bible-template.md` を埋める。
8. `brand-setting`, `brand-production-brief`, `brand-system-prompt`, 工程別 prompt 束へ、固定IP条件を転記する。
9. rough / final / export-ready の各 gate で、IP固定条件が保たれているか確認する。
10. drift が見えたら、原因を `構造 / ブランド / 商品 / 権利 / 制作パイプライン` に分け、戻し先を決める。

## キャラクターIPで必ず固定すること
- 顔の比率、目口の形、輪郭、身体比率、代表ポーズ。
- 反応温度、口調、得意な感情、不自然な感情。
- 同一キャラクターとして認識できる最低要素。
- 別キャラに見える境界。

## 記号・モチーフIPで必ず固定すること
- 基本シルエット、線の文法、余白、反復単位。
- 残すディテールと捨てるディテール。
- アイコン、ロゴ、UI部品へ寄りすぎない境界。

## Gate
次のどれかが欠ける場合、固定IPとしては `Revise` とする。
- 権利の所在が明確である。
- 代表シルエットが小表示でも識別できる。
- 変えてよい範囲と変えてはいけない範囲が分かれている。
- rough と final の両方で同じIPとして見える。
- 量産時の追加差分が、別IP化せず増やせる。
- 販売 metadata とIPの見え方が矛盾していない。

## 出力原則
- 世界観設定より、制作で守れる固定条件を優先する。
- 似ている既存IPや素材依存がある場合は、魅力より先に権利リスクを扱う。
- 固定しすぎて release ごとの商品差分が作れない状態を避ける。
