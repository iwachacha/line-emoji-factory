This file records the factory's current operational interpretation of LINE Creators Market emoji and sticker requirements. When official requirements change, update this file before changing validators or templates.

## Product Scope Baseline

- A brand may support more than one LINE item type.
- A release chooses one concrete item type. Do not require every brand to release both emoji and stickers.
- Supported packageable item types in this repo:
  - `static-emoji`
  - `static-sticker`
- Animation validation exists for `animation-emoji`, but release packaging remains unsupported.
- Animated stickers, custom stickers, message stickers, big stickers, pop-up stickers, and effect stickers are outside the current packaging contract.

## Static Emoji Baseline

- Tab image: 1 file, `96 x 74` px.
- Content images: `8`, `16`, `24`, `32`, or `40` files.
- Content image size: `180 x 180` px.
- Image format: PNG.
- Background: transparent.
- Resolution: at least 72 dpi.
- Color mode: RGB-compatible.
- One image file: `1MB` or less.
- ZIP upload: `20MB` or less.

## Static Sticker Baseline

- Main image: 1 file, `240 x 240` px.
- Sticker images: `8`, `16`, `24`, `32`, or `40` files.
- Sticker image size: up to `370 x 320` px.
- Chat thumbnail icon: 1 file, `96 x 74` px.
- Image format: PNG.
- Background: transparent.
- Resolution: at least 72 dpi.
- Color mode: RGB-compatible.
- One image file: `1MB` or less.
- ZIP upload: `60MB` or less.
- Sticker image width and height should be even numbers because LINE resizes sticker images automatically.
- A margin of around `10px` is expected between trimmed content and the image edge.

## Animation Emoji Baseline

- Content images use APNG files with `.png` extension.
- Content APNG size: `180 x 180` px.
- One APNG file: `300KB` or less.
- Frame count: `5` to `20`.
- Total duration: `4` seconds or less.
- Loop count: `1` to `4`.
- The static tab image remains `96 x 74` px PNG.
- Static packaging remains the only supported package flow in this repo until animation release packaging is added.

## Metadata Baseline

- Creator name: 50 LINE-count characters or fewer.
- Emoji or sticker title: 40 LINE-count characters or fewer.
- Description: 160 LINE-count characters or fewer.
- Copyright: 50 LINE-count characters or fewer.
- Copyright uses ASCII letters and numbers only.
- Metadata must not include emoji characters.

## Sales Operation Baseline

- Items are submitted with text and required images, reviewed against LINE sales criteria, then released for sale after approval.
- Manual release is the default sales setting. Approved items can also be configured to start sales automatically.
- Items cannot be put on sale before review approval.
- Items cannot be distributed for free through Creators Market; buyers generally need to purchase them before use.
- Available price tiers are selected from LINE's options and may change; confirm final price in LINE STORE or the in-app shop before release.
- Revenue share is transferred after platform transaction fees according to LINE Creators Market rules.
- Payment requests require the revenue share threshold and registered transfer information.
- LINE Stickers Premium eligibility is an operating decision, not a reason to force both emoji and sticker releases.

## Factory Filename Interpretation

- Production filenames are internal and may be arbitrary.
- Submission filenames are generated by `tools/package-release.py`.
- Regular static emoji submission content images use zero-padded numeric names: `001.png`, `002.png`, ... up to the active set count.
- The filename numbering contract differs for kana / letter emoji package types; this factory currently supports regular emoji packaging only.
- The tab image is generated as `tab.png`.
- Static sticker submission content images use two-digit numeric names: `01.png`, `02.png`, ... up to the active set count.
- Static sticker main image is generated as `main.png`.
- Static sticker chat thumbnail icon is generated as `tab.png`.
- The LINE upload ZIP must contain only image files.

## Review-Risk Interpretation

- Promotional terms such as campaign, free, giveaway, sale, and discount are warnings.
- Brand, structure, and product fit must still be judged in the relevant rules and workflows.
