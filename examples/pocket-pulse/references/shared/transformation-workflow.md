# 変換ワークフロー

このファイルは、構造で `Fail` した案、または商品化不成立の案を、
成立方向へ変換する標準手順を定義する。

## 起動条件
- `rules/structure-constraints.md` で `Fail`。
- 構造は通るが、`商品転換力` が弱く商品化できない。
- 面白さや世界観はあるが、単体使用や日常使用へ落ちない。

## 手順
1. 何が `Fail` 要因かを1文で特定する。
2. 元案の核を捨てるべきか、構造だけ変えるべきかを判定する。
3. 少なくとも2つの変換方向を出す。
4. 各変換案に、新しい `構造分類` を付ける。
5. 変換後の案を、再度 `rules/structure-constraints.md` で確認する。
6. まだ `Fail` なら切る。

## 標準パターン
### 文字列を囲みたい
- 左パーツ / 右パーツへ分解する。
- 囲み感ではなく、前置き / 後置き演出へ変える。
- デコ文字ブランドへ再定義する。

### 文字に下線を引きたい
- 下線ではなく、強調記号や添え記号へ置き換える。
- 記号型ブランドとして再設計する。

### 世界観は強いが使い道が弱い
- 汎用反応語彙へ落とす。
- 日常温度帯を追加する。
- 文末補助の用途を足す。

### デコ文字化したいが装飾が重い
- 装飾量を減らす。
- 崩れやすい文字だけ例外ルールを持つ。
- 一律デザインをやめ、可読性優先で整える。

### 面白いが単体で意味が弱い
- 単体送信時の意味を先に定義する。
- 各アイテムに用途ラベルを与える。

## 出力フォーマット
- `Fail要因`
- `残す核`
- `変換案A`
- `変換案B`
- `変換後の構造分類`
- `まだ残るリスク`
