# そえしるし ブランド固有制作基盤

この文書は、`そえしるし` を実制作へ接続する正本である。
ブランド核は [brand-setting.md](./brand-setting.md) を優先し、
ここでは制作工程と商品運用の固定条件だけを扱う。

## 1. 基本情報
- ブランド名: `そえしるし`
- 参照するブランド設定: `brand/brand-setting.md`
- 判定状態: `Design Ready`
- 商品化の目的: 文字主体の会話で使える、軽量な強調・整理用絵文字の定番セットを作る。
- 商品コンセプト: 重要度・依頼・状態・締めを、文字の脇で軽く整理するコア実用品セットを作る。
- ブランド内での役割: `そえしるし` の基準語彙をまとめた最初のコア商品。後続のやわらか版、推し活版、用途特化版の母集合になる。

## 2. 公式仕様の確定
- 商品種別: `静止絵文字`
- パッケージタイプ: `絵文字`
- 初期セット個数: `16`
- 初期セット個数の根拠: `8` では構造の最小証明にはなるが、商品コンセプトである `入口・依頼・状態・締めを1商品で整理する` には語彙が足りない。`16` なら主力14枠 + 組み合わせ補助2枠で、初回購入価値と差分明確性を両立できる。
- 必須画像: トークルームタブ画像 1個、コンテンツ画像 16個
- 画像形式: `PNG`
- 画像サイズ: タブ画像 `96 x 74 px`、コンテンツ画像 `180 x 180 px`
- 容量制約: 画像1個あたり `1MB以下`、ZIP合計 `20MB以下`
- 背景透過: 必須
- メタデータ制約: タイトル40文字以内、説明文160文字以内、コピーライト50文字以内、コピーライトは英数字のみ

## 3. 固定制作パイプライン
- rough stage の目的: 機能ごとのシルエット差と、文字の脇役としての温度感を先に固定する。
- rough stage で先に固定するもの: 外形シルエット、中央サイン、左右ペアの向き、上段4個の優先度、主力枠と補助枠の強弱、色の役割分担。
- rough board / 全体図の最小構成: セット一覧、上段4個の拡大ラフ、左右ペアの並置、主力枠と補助枠の見え方比較、外形だけの白黒比較、差分軸メモ。
- rough stage の成功条件: 小表示でも各機能の差が読めること、左右ペアが単体でも未完成に見えないこと、色を抜いても判別できること、主力枠が補助枠に食われていないこと。
- ClaudeDesign へ渡す単位: `release-001` 全体条件 + 各絵文字の個別仕様。
- ClaudeDesign で仕上げる範囲: 線の整理、角の丸み統一、余白調整、機能サインの視認性調整、タブ画像整合。
- 仕上げで変えてよいもの: 角の精度、太さの微調整、色味の濃淡、中央サインの大きさ。
- 仕上げで変えてはいけないもの: 1機能1外形の対応、左右ペアの方向、脇役性、過装飾禁止。
- 差し戻しルール: 構造が崩れたら構造再評価へ戻す。差分が弱いなら rough 再設計。線の整理不足だけなら Stage 2 で修正する。

## 4. 表示前提
- 単体送信時の役割: `重要`, `確認`, `お願い`, `共有`, `見て`, `注意`, `補足`, `要返信`, `了解`, `決定`, `あとで`, `締切`, `メモ`, `おすすめ`, `結論` などの短い意図を単独で伝える。
- 文中使用時の役割: 単語や短文の前後に添えて、視線誘導や意味ラベルとして機能する。
- 複数絵文字連続時の役割: 2個までの見出し化、または `みて左 + テキスト + みて右` の短い囲み補助に限定する。組み合わせ用途は補助枠に閉じ込める。
- 余白方針: 原則少なめ。内向き側だけ潰れを避ける余白を少し残し、文字と接したときの息苦しさを減らす。
- アウトライン方針: 濃いダークブラウン1本で太く統一する。細線や二重線は禁止。
- 装飾方針: 機能サイン以外の飾りは足さない。きらめき、集中線、紙吹雪は不採用。

## 5. セット構成
- 最上段に置く高頻度絵文字: `重要`, `確認`, `お願い`, `共有`
- 基本反応語彙: 重要、確認、お願い、共有、注目、注意、補足、要返信、了解、決定、あとで、締切、メモ、おすすめ、結論
- 差分の軸: 機能、向き、角の切れ方、中央サインの種類、開閉方向
- 主力枠と補助枠の配分: 主力枠 `14`、組み合わせ補助枠 `2`
- バリエーション方針: まず実用品の基本機能を揃え、その後に温度差分や用途特化を足す。
- 並び順方針: 上段は高頻度の独立用途、2段目に導線補助と注意・補足、3段目に依頼応答と状態、4段目に締めや軽い日常用途を置く。
- 重複禁止ルール: 色違いだけは不可。左右ミラー差分は `みて左 / みて右` の1組まで。意味が近いものは中央サインか外形で明確に差を作れない限り追加しない。

## 6. stage 別制作ルール
- 初期 release spec の作成ルール: 各絵文字に `用途 / 単体送信時の意味 / 文中使用時の役割 / 連続使用メモ` を必ず書く。
- rough stage で使う視覚 anchor: 角丸フック、太枠、中央サイン1個、外形ごとの大きな差、開き方向の違い。
- rough stage で未確定にしてよい点: アクセント色の微調整、影の有無、角の丸みの細かい数値。
- ClaudeDesign に必ず渡す共通条件: 文字より前に出すぎないこと、線を細くしないこと、中央サインを潰さないこと、色がなくても機能差が読めること、主力枠が補助枠より先に読めること。
- 1絵文字ごとの完成条件: 小表示で用途が読めること、単体でも未完成に見えないこと、セット内の別絵文字と取り違えないこと。
- ファイル命名 / バージョン方針: `soe-001-{id}-v{nn}.png`。rough と final を分離し、版上げは2桁。
- release handoff の作成ルール: 各絵文字ごとに `keep / strengthen / trim / risk` を1行で明記する。

## 7. デコ文字・組み合わせ・アニメーション個別方針
- デコ文字可読性ルール: 該当なし
- 組み合わせ使用ルール: `みて左 + テキスト + みて右` の短い囲み補助だけを明示採用。4個以上の連鎖や長文の囲みは非推奨。
- 組み合わせ補助枠の扱い: `みて左 / みて右` の2枠のみ採用する。高頻度上段の多数派にせず、他枠は単独機能記号として設計する。
- 1フレーム目ルール: 該当なし。静止絵文字で運用する。
- アニメーションの見せ方: 初期 release では不採用
- フレーム / 再生時間 / ループ設計メモ: 該当なし

## 8. 登録・販売運用
- タブ画像方針: `そえしるし` の外形文法が一目で分かる、左右ペアの中間的な記号を1個置く。
- タイトル方針: 文字補助・強調・整理の実用性が読める短いタイトルにする。
- 説明文方針: 「重要・確認・お願い・共有を軽く整理できる」「重要語の前後に添えやすい」を明記する。
- コピーライト表記: `SOESHIRUSHI`
- サジェスト表示タグ候補: `重要`, `注意`, `確認`, `メモ`, `締切`, `結論`, `見る`, `強調`
- リリース前チェック: 左右ペアの単体成立、`補足` と `結論` の差、`確認 / 要返信 / 了解` の差、上段4個の妥当性、メタデータと実機能の一致を重点確認する。

## 9. 審査注意
- Hard NG 回避項目: 他社ロゴ連想、UIアイコンの模倣、誤解を招く危険表示、仕様不一致
- Revise になりやすい項目: タイトルと機能のズレ、見た目が地味すぎて用途が読めない、左右ミラー差分が水増しに見えること
- Watch 項目: 左右ペアが「片割れ」に見えるリスク、色に頼りすぎる差分、`補足` と `結論` の混同、主力枠が仕事寄りに寄りすぎること
- 品質台帳へ残すべき論点: ペア単体成立、差分明確性、上段4個の頻度妥当性、主力14枠のバランス、タブ画像と本体の整合
- 継続監査で確認する仮説: `確認 / 要返信 / 了解` は実会話で向き差が読める、`おすすめ` は実用品 set 内でも浮きすぎず買う理由になる、`共有 / メモ` は文中使用で役割差が残る、左右ペアは補助枠のまま主力を食わない

## 10. 量産・拡張
- 次に増やす差分: `開始`, `しめ`, `集合`, `報告`, `ネタバレ注意`, `やわらかお願い`
- 季節派生の可否: 可。ただし季節装飾は別 release に切り、常設の実用核は崩さない
- 実用品派生の可否: 高い。仕事連絡版、推し活版、学習メモ版へ展開可能
- プレミアム / 長期運用の見立て: 通年用途があるため長期運用向き
- 商品化・ライセンス拡張メモ: ノートアプリ風の外部サービス連想は避け、あくまで汎用記号ブランドとして育てる
- factory へ戻す改善候補: 実用品 set 内で `おすすめ` のような温度違い枠が浮く条件、左右補助枠の単体成立を検証する最小粒度
- 運用を重くしないためのルール: usage validation は decisive な 6 シーン前後に絞る。 retrospective は milestone ごとに 1 本だけ維持し、閉じた迷いは quality ledger へ戻さない
