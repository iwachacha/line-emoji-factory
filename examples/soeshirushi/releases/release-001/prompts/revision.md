# Revision Prompts

## Review Fixes
- `審査で左右ペアが片割れと判断された場合、単体でも完結して見えるように外形を調整し、ペア前提の欠けた形を避ける`
- `ロゴ風と判断された場合、幾何の規則性を崩しすぎずに固有マーク感を弱め、実用記号へ戻す`

## Quality Fixes
- `visibility: 小表示で中央サインが読めない場合、外形は維持したままサインを大きくし、余白を削って調整する`
- `difference clarity: 補足 / 結論 が近い場合、片方を開く形、片方を閉じる形へ振り分ける`
- `direction clarity: 確認 / 要返信 / 了解 が近い場合、確認は止める、要返信は戻す、了解は収める方向の中央サインへ振り分ける`
- `spacing: 文中で詰まって見える場合、内向き側の余白だけ少し足し、外周は大きく変えない`
- `metadata alignment: タイトルや説明文が実用品感を伝えきれていない場合、文字補助・強調・整理の語を優先する`

## Post-Release Fixes
- `quality-ledger の Watch を見て、色依存差分が見つかったらシルエット差へ置き換える`
- `日常接続性が弱いと判明したら、次 release で仕事寄り機能を減らし、日常語の補助記号を増やす`
- `補助枠が強すぎると判明したら、次 release では主力枠を増やし、組み合わせ補助枠は据え置く`
