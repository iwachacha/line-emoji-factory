# release-001 制作ハンドオフ

この handoff は、`release-001 ことば整理コア16` を
rough 設計から final 仕上げへ渡すための正本である。

## 1. 基本情報
- ブランド名: `そえしるし`
- release / set 名: `ことば整理コア16`
- 参照するブランド設定: `brand/brand-setting.md`
- 参照する制作基盤: `brand/brand-production-brief.md`
- 参照する set 仕様: `releases/release-001/release-spec.md`
- 参照する quality ledger: `releases/release-001/qa/quality-ledger.md`
- 参照する usage validation: `releases/release-001/qa/usage-validation.md`
- 判定状態: `Design Ready`

## 2. rough stage の要約
- rough stage の目的: 16個の機能差分と、脇役としての統一感を固める
- 使用した rough board / 全体図: `releases/release-001/releases/release-001/production/rough-boards/release-001-board-notes.md`
- 固定できた視覚記号: 角丸フック系外形、中央サイン1個、濃いアウトライン、内向きの抜け
- 固定できた差分軸: 機能、向き、中央サイン、外形の切れ方、開閉方向
- まだ未確定の点: 色味の最終調整、`補足 / 結論` と `確認 / 要返信 / 了解` の中央サインの細部
- item finalization に渡す理由: ラフ段階で差分の筋は見えたため、ここからは小表示の精度、主力枠と補助枠のバランス、セット整合性を詰める段階に入る

## 3. set 全体で守ること
- 単体送信時の成立条件: 1個だけ送っても `重要`, `確認`, `お願い`, `共有`, `見て`, `注意`, `補足`, `要返信`, `了解`, `決定`, `あとで`, `締切`, `メモ`, `おすすめ`, `結論` のいずれかが読めること
- 文中使用時の成立条件: テキストの前後に置いても、文字を覆わず、視線誘導か意味ラベルとして自然に働くこと
- 連続使用時の成立条件: 2個までならノイズにならず、`みて左 + テキスト + みて右` は短い語だけで成立すること。組み合わせ用途が主力枠を食わないこと
- 絶対に残す要素: 外形の太さ、中央サイン1個、角丸の文法、脇役性
- 削ってよい要素: 補助的な陰影、角の細かなニュアンス、二次装飾
- 禁止 drift: キャラ化、ロゴ化、下線帯化、囲み枠の完全化、色でしか差が読めない設計、補助枠の過剰増殖
- 余白 / アウトライン / 装飾の補足: 余白は少なめだが内向き側だけ少し残す。アウトラインは全絵文字で同じ強さ。装飾追加は禁止。

## 4. item finalization への共通指示
- 仕上げの目的: 小表示の可読性、差分明確性、セット全体の輪郭統一を仕上げること
- 1絵文字ごとに優先する順序: `用途の読め方 → シルエット差 → 中央サインの視認性 → 細部の美しさ`
- rough からの変更許容範囲: 線の整理、角の調整、色の濃淡変更、中央サインの拡大縮小
- 変更してはいけない核: 1機能1外形、脇役性、左右ペアの向き、過装飾禁止
- 小表示で最優先する点: 外形と中央サインだけで機能が判別できること
- set 全体で揃える点: 太さ、角丸、アウトライン色、脇役としての落ち着き、主力枠と補助枠の強弱
- 未解決 `Watch` 項目: 左右ペアの単体成立、`補足 / 結論` の見分け、`確認 / 要返信 / 了解` の見分け、`共有 / メモ / おすすめ` の勝ち筋、上段4個の頻度妥当性
- 生成後に優先監査する絵文字: `S05`, `S06`, `S08`, `S09`, `S10`, `S14`, `S15`, `S16`
- タブ画像と metadata への注意: タブ画像は単一記号でブランド核を見せる。metadata では「文字補助」と「実用性」を言い切る。

## 5. 個別絵文字仕様
- `S01 / 重要`
  - 用途: 強い重要表示
  - 単体送信時の意味: `大事`
  - 文中使用時の意味: 最重要文の前置き
  - 連続使用メモ: 原則1個で使う
  - 残す rough 要素: 太い封緘風外形
  - 仕上げ時に強める点: `重要` の重み
  - 仕上げ時に削る点: 単なるびっくり記号感
  - 注意する審査 / リスク: `注意` と混ざらないこと
- `S02 / 確認`
  - 用途: 確認依頼
  - 単体送信時の意味: `確認して`
  - 文中使用時の意味: 要確認項目の頭
  - 連続使用メモ: 箇条書きにも使える
  - 残す rough 要素: 止める印象の中央サイン
  - 仕上げ時に強める点: 中央サインの明瞭さ
  - 仕上げ時に削る点: 余計な角の飾り
  - 注意する審査 / リスク: `要返信` や `了解` と混ざらないこと
- `S03 / お願い`
  - 用途: 依頼、お願い
  - 単体送信時の意味: `お願いします`
  - 文中使用時の意味: 依頼文の頭
  - 連続使用メモ: `確認` と並べる場合も差が必要
  - 残す rough 要素: やわらかいタブ形
  - 仕上げ時に強める点: 圧を出しすぎない依頼感
  - 仕上げ時に削る点: 感情的な可愛さ
  - 注意する審査 / リスク: `共有` や `要返信` と混ざらないこと
- `S04 / 共有`
  - 用途: 情報共有、周知
  - 単体送信時の意味: `共有です`
  - 文中使用時の意味: 周知文の頭
  - 連続使用メモ: `重要` と並ぶ場合も意味差が必要
  - 残す rough 要素: 外へ開く共有サイン
  - 仕上げ時に強める点: 拡散より伝達のニュアンス
  - 仕上げ時に削る点: SNSアイコン連想
  - 注意する審査 / リスク: 他社サービス連想を避けること
- `S05 / みて左`
  - 用途: 左からの注目導線
  - 単体送信時の意味: `見て / ここ`
  - 文中使用時の意味: 重要語の左に置いて導入を作る
  - 連続使用メモ: `S06` と対で使える
  - 残す rough 要素: 大きい内向きフック、中央の注目サイン
  - 仕上げ時に強める点: 単体での完結感
  - 仕上げ時に削る点: 内側の細い補助線
  - 注意する審査 / リスク: 片割れ感を残さない
- `S06 / みて右`
  - 用途: 右からの注目導線
  - 単体送信時の意味: `ここ / 注目`
  - 文中使用時の意味: 重要語の右に置いて締めを作る
  - 連続使用メモ: `S05` と対で使える
  - 残す rough 要素: 反転した外形、中央の注目サイン
  - 仕上げ時に強める点: 単独使用時の違和感解消
  - 仕上げ時に削る点: 補助的な尾の長さ
  - 注意する審査 / リスク: `S05` との差が向き以外でも読めるか
- `S07 / 注意`
  - 用途: 注意喚起
  - 単体送信時の意味: `注意`
  - 文中使用時の意味: 危険ではないが気をつける項目の前置き
  - 連続使用メモ: 1個で立てる
  - 残す rough 要素: 警戒感のある角の切れ
  - 仕上げ時に強める点: `重要` との違い
  - 仕上げ時に削る点: 危険標識に見える鋭さ
  - 注意する審査 / リスク: 道路標識や既存安全マークへの接近
- `S08 / 補足`
  - 用途: 追記と備考
  - 単体送信時の意味: `補足あり`
  - 文中使用時の意味: 本文後の追記ラベル
  - 連続使用メモ: 1個のみ
  - 残す rough 要素: 折り返しメモ風の外形
  - 仕上げ時に強める点: 柔らかい追記感
  - 仕上げ時に削る点: `結論` と似る閉じた形
  - 注意する審査 / リスク: `結論` と混同されないこと
- `S09 / 要返信`
  - 用途: 返信依頼、返答待ち
  - 単体送信時の意味: `返信ください`
  - 文中使用時の意味: 返答が必要な文の頭
  - 連続使用メモ: `お願い` と並べても役割差を保つ
  - 残す rough 要素: 戻る印象の中央サイン
  - 仕上げ時に強める点: `確認` との差
  - 仕上げ時に削る点: 催促の圧
  - 注意する審査 / リスク: `確認` と混ざらないこと
- `S10 / 了解`

  - 単体送信時の意味: `了解`
  - 文中使用時の意味: 了承や受領の返答
  - 連続使用メモ: 1個で完結
  - 残す rough 要素: 収める印象の中央サイン
  - 仕上げ時に強める点: `確認` や `要返信` と違う落ち着き
  - 仕上げ時に削る点: 固すぎる印象
  - 注意する審査 / リスク: 方向差が弱いと混ざること
- `S11 / 決定`
  - 用途: 決まり、確定
  - 単体送信時の意味: `決定`
  - 文中使用時の意味: 決定事項の頭
  - 連続使用メモ: `結論` と併用する場合も役割差を保つ
  - 残す rough 要素: 閉じる印象の強い外形
  - 仕上げ時に強める点: 確定感
  - 仕上げ時に削る点: 重すぎる威圧感
  - 注意する審査 / リスク: `結論` と混ざらないこと

  - 文中使用時の意味: 今は保留の項目の頭
  - 連続使用メモ: `メモ` と近づきすぎないこと
  - 残す rough 要素: 保留の抜け感
  - 仕上げ時に強める点: 放置ではなく保留の意味
  - 仕上げ時に削る点: 緩すぎる印象
  - 注意する審査 / リスク: `メモ` との差
- `S13 / 締切`
  - 用途: 期限と急ぎ
  - 単体送信時の意味: `急ぎ / 期限`
  - 文中使用時の意味: 日時や締切の前置き
  - 連続使用メモ: `重要` と並べる場合は片方を弱める
  - 残す rough 要素: 旗やピンに近い外形
  - 仕上げ時に強める点: 時間制約の気配
  - 仕上げ時に削る点: 赤頼みの強さ
  - 注意する審査 / リスク: 危険警告と誤読されないこと
- `S14 / メモ`
  - 用途: メモ、控え、記録
  - 単体送信時の意味: `メモ`

  - 連続使用メモ: `共有` や `補足` と差が必要
  - 残す rough 要素: 記録感のある外形
  - 仕上げ時に強める点: 控えのニュアンス
  - 仕上げ時に削る点: 説明過多な紙感
  - 注意する審査 / リスク: `共有` や `補足` と混ざらないこと

  - 注意する審査 / リスク: `重要`・`補足` の中間に落ちないこと

## 6. 出力管理
- 出力ファイル命名方針: `soe-001-{id}-v{nn}.png`
- バージョン管理方針: rough は `r`, final は `f` を付けて管理し、差し戻しごとに版を上げる
- rough / final の保存先: `releases/release-001/production/rough-boards/`, `releases/release-001/production/finals/`
- 修正依頼の記録先: `releases/release-001/qa/quality-ledger.md`

## 7. 差し戻し条件
- Stage 2 で直す問題: 線の太さ、余白、中央サインの潰れ、色の濃淡
- Stage 1 へ戻す問題: 機能差が弱い、左右ペアが単体で成立しない、`確認 / 要返信 / 了解` の方向差が弱い、上段4個の優先順位が揺れる
- 構造再評価へ戻す問題: 囲み前提でしか意味が成立しない、単体送信で読めない、配置依存が強すぎる
