# Visual Asset Quality Rules

このファイルは、LINE絵文字 / スタンプの実画像を商品品質として判断するための正本である。
`rules/asset-validation-rules.md` は機械検査と公式仕様の gate、このファイルは目視・商品 QA の gate を扱う。

## 判断順序
1. 構造: 1アイテム単体で意味と使用場面が成立しているか。
2. ブランド: brand canon と series plan を壊していないか。
3. 商品: LINE上の小表示と会話投入で使えるか。

## 必須条件
- 背景は透明にする。白背景、色ベタ背景、写真背景は原則 `Hard NG`。
- アウトラインは太く、濃い色を基本にする。
- 余白は少なく、主情報をキャンバス内で大きく置く。
- 小表示では、顔、手、文字、記号、ポーズのうち主情報だけが残れば読める設計にする。
- 装飾は主情報を助ける範囲に限る。
- 1アイテムで意味、用途、反応温度が伝わる。
- 表情差分またはポーズ差分は、48px 相当でも見分けられるほど大きくする。
- セット全体で用途が被りすぎない。
- シリーズ内で継承要素と新規性要素が同時に読める。

## 小表示視認性
- `180px`: 公式サイズで主線、余白、透過、装飾密度を見る。
- `96px`: タブや一覧プレビューに近い縮小で silhouette と色塊を見る。
- `48px`: 会話内で即読できるかを見る。
- `32px`: 絵文字の最小運用に近い縮小で、主要記号だけが残るかを見る。

`48px` と `32px` で主要要素が消える場合は、絵柄がかわいくても `Revise` 以上に上げない。

## 装飾上限
- 背景、小物、集中線、キラキラ、粒、柄、複数の表情記号を同時に盛らない。
- 読ませたい文字や顔より装飾が先に見える場合は `Revise`。
- 商品説明、世界観説明、長文、細い小物で意味を補う設計は `Hard NG` 寄りに扱う。

## シルエット優先
- 色を抜いても、おおよその用途や感情が残ること。
- 外形差、顔の向き、手の位置、ポーズ、記号の形を差分軸にする。
- 色違いだけ、微妙な表情差だけ、同じポーズの言い換えだけで水増ししない。

## 1アイテム意味条件
各アイテムは、最低限以下を説明できること。
- 用途
- 単体送信時の意味
- 文中使用時の意味
- スタンプ送信時の発話
- 小表示で残す視覚記号
- 削ってよい装飾
- 禁止 drift

説明できない場合は、画像品質ではなく商品設計の `Revise` として扱う。

## セット内重複
- 同じ用途、同じ感情温度、同じポーズ、同じ記号が複数ある場合は用途被りとして確認する。
- 重複に見えるアイテムは、片方を削る、用途を変える、表情/ポーズ/文字/記号を大きく変える。
- 近似画像が残る場合は、過去商品との差分またはシリーズ内での役割を quality ledger に残す。

## 不合格例
- 背景がある。
- 線が細い。
- 主情報が小さく余白が広い。
- 細部、小物、装飾が多すぎる。
- 淡色だけで低コントラスト。
- 表情差分が弱い。
- ポーズ差分が弱い。
- 似た差分が並ぶ。
- 使い道が不明。
- 単体送信で意味が死ぬ。
- 文中使用で邪魔になる。
- スタンプ送信で発話として弱い。
- セット全体の統一感がない。
- 過去商品とほぼ同じで新シリーズの理由がない。
- 新鮮さを出すために brand canon / IP guardrails を壊している。

## QA 記録
- `Hard NG`: 提出しない。理由を release checklist の blocking 項目へ戻す。
- `Revise`: 再生成または item spec 修正。失格理由を quality ledger に残す。
- `Watch`: 採用可能だが次 release で再発を見る。brand product catalog にも継承/回避メモを残す。
