# 申請 metadata ルール

このファイルは、LINE申請前に creator、title、description、copyright を検査する判断基準の正本である。

## 公式仕様から守ること
- クリエイター名は `50文字以内`。
- 絵文字 / スタンプタイトルは `40文字以内`。
- 絵文字 / スタンプ説明文は `160文字以内`。
- コピーライトは `50文字以内`。
- コピーライトは英数字のみ。
- 全角文字は `2文字` として数える。
- metadata に絵文字は使わない。

## factory で止める表現
- 販売開始、割引、キャンペーンなど告知が主目的の表現。
- LINE以外のサービス名を主用途として押し出す表現。
- 他社キャラクター名、商標、作品名に依存した表現。
- 特定個人だけに向いた名前入り表現。
- 単なる企業ロゴや告知素材に見える表現。
- 絵文字 / スタンプ内容と title / description が一致しない表現。

## 判定
- `Hard NG`: 文字数超過、copyright 不正、絵文字混入、明確な審査NG表現。
- `Revise`: 内容と metadata のずれ、訴求の弱さ、汎用性不足。
- `Watch`: 通過見込みはあるが、審査差し戻し時に記録へ戻す表現。

## tool 接続
- 最小機械検査は `tools/validate-metadata.py` で行う。
- 申請前の総合判断は `workflows/submission-audit-workflow.md` に接続する。
