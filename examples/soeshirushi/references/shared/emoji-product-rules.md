# 絵文字商品ルール

このファイルは、ブランドを実際のLINE絵文字商品へ落とす際の判断基準を扱う。
商品品質に関する項目は、このファイルで扱う。

## 公式仕様との接続
- 技術仕様と表示事実は `rules/line-platform-baseline.md` を前提にする。
- このファイルでは、公式制作ポイントを商品判断へ翻訳して扱う。

## 商品層で必ず見る項目
- `視認性`
- `即読性`
- `両用性`
- `低摩擦性`
- `再使用性`
- `差分明確性`
- `セット整合性`
- `連続使用適性`
- `量産性`
- `商品転換力`

## 商品設計の原則
- 小表示で主要要素が落ちない。
- 文末に添えても浮きすぎない。
- 単体送信でも成立する。
- 単体送信 / 文中使用 / 2個連続の3パターンを、短い会話サンプルで検証できる。
- セット内に実質重複を作らない。
- 色違いだけ、微差分だけで差分を水増ししない。
- ブランドの核が、視覚記号と用途へ自然変換されている。
- 初期セット個数は `公式最小数だから` ではなく、商品コンセプト、差分の軸、再使用場面の幅に見合う根拠で決める。
- release / set ごとに `この商品を買う理由` を一言で言える状態にする。これが弱いと `セット整合性` と `商品転換力` が落ちる。
- 組み合わせ寄りの補助絵文字を少量含めること自体は許容する。ただし set の多数派や上段高頻度枠を占めさせず、各パーツ単体の意味を残す。

## 公式制作ポイントの運用化
- 余白は原則少なくし、絵柄を大きく使う。
- ただしデコ文字・テキスト絵文字は、可読性確保のため少しだけ余白を持たせる。
- アウトラインは太く、濃い色を基本にする。
- 表情や反応差分は、小表示でも分かるほど大きくする。
- 装飾は最小限にし、キラキラや集中線で情報を埋めない。
- セット上部には高頻度用途を置く。
- 構図やポーズに変化を入れ、パッケージ全体を単調にしない。
- アニメーションを使うなら、1フレーム目だけでも意味が読め、動き自体も視認できることを求める。

## 構造分類ごとの最低合格条件
### 単体反応型
- 各絵文字が単体で意味を持つ。
- `再使用性` を支える基本反応語彙がある。

### 組み合わせ型
- 各パーツ単体でも最低限の意味が残る。
- 並べ方依存が強すぎて、単体使用が死なない。
- 補助枠として採る場合も、`単体で弱いが演出で便利` を免罪符にしない。

### デコ文字型
- 可読性を最優先する。
- 文字ごとの崩れやすい箇所に例外ルールを持つ。
- 文字同士が詰まりすぎない余白運用を持つ。

### 混合型
- 主構造が明示されている。
- 主構造側の最低合格条件を満たしている。

## ブランド主型ごとの最低合格条件
### キャラクター
- 同一ブランド内で人格と反応温度がぶれない。
- 表情差分が `差分明確性` を生み、ノイズにならない。

### モチーフ
- モチーフが小表示でも識別できる。
- 残すディテールと捨てるディテールが決まっている。

### 記号
- 説明なしでも用途が読める。
- 装飾過多で `即読性` を落とさない。
- 企業ロゴや広告記号に寄りすぎない。

### 図形・抽象
- 造形文法が揃っている。
- 抽象性が高くても、反応や用途が無に落ちない。

### デコ文字
- 文字が主役であることを崩さない。
- 記号性を足しても可読性を犠牲にしない。

## 補助タグ付きの最低合格条件
### 実用品・機能
- 高頻度シーンが明示できる。
- 既存の汎用表現と比べた実用優位点がある。

### 季節・文化
- 使用期間または使用文脈が定義されている。
- シーズン外に使えない場合は、通年派生方針がある。

## ブランド固有基盤への接続
- この工場では、制作方法を `GPT-image2.0 ラフ生成 → ClaudeDesign 仕上げ` で固定する。
- `Design Ready` の後、制作実装へ進める場合は `workflows/production-pipeline-workflow.md` で工程責務を固定する。
- `Design Ready` の後、制作実装へ進める場合は `templates/brand/brand-production-brief-template.md` を埋める。
- 初期 release / set 設計は `templates/release/release-spec-template.md` を埋める。
- release / set 単位の引き継ぎは `templates/release/production-handoff-template.md` を埋める。
- 提出前と継続改善は `workflows/quality-control-workflow.md` と `templates/qa/release-checklist-template.md`, `templates/qa/quality-ledger-template.md` を使う。
- `brand-production-brief` では、推奨パッケージタイプ、初期個数、差分配列、余白方針、タグ方針、審査注意、工程責務を固定する。

## 判定の目安
- `Revise`
  - 用途はあるが、視認性・差分明確性・セット整合性のいずれかが弱い。
  - 初期セット個数の根拠や release 固有コンセプトが弱く、商品として買う理由が薄い。
- `Design Ready`
  - セット構成、差分設計、量産方針、rough stage と仕上げ stage の接続方針、初期 release 設計の筋まで明文化できる。
