# release-001 セット仕様

この release は、`そえしるし` の最初のコア商品である。
「文字の脇で、重要度・依頼・状態・締めを整理する」という商品コンセプトを、16個で明確に示す。

## 1. 基本情報
- ブランド名: `そえしるし`
- release / set 名: `ことば整理コア16`
- 参照するブランド設定: `brand/brand-setting.md`
- 参照する制作基盤: `brand/brand-production-brief.md`
- 判定状態: `Design Ready`
- 商品種別: `静止絵文字`
- 初期セット個数: `16`
- 初期セット個数の根拠: `8` では構造の最小証明にはなるが、入口・依頼・状態・締めを1商品で整理するには語彙が足りない。`16` なら主力14枠 + 組み合わせ補助2枠で、初回購入価値とセット差分を両立できる。
- この release の一言コンセプト: 伝える前後を軽く整える、文字補助のコア16。
- ブランド内での役割: `そえしるし` の基準語彙を定義する最初のコア商品。後続派生の比較基準になる。

## 2. release の目的
- この release で取りにいく使用場面: 重要語の前後、確認依頼、お願い、共有、短い注意書き、補足、予定調整、結論提示
- 主な相手 / 関係性: 自分メモ、同僚、チーム連絡、友人との予定調整、推し活の要点共有
- 先に解くべき会話課題: 記号だけだと冷たく、スタンプだと重い場面で、短文を整理しながら軽く意味を足せる選択肢が少ない
- 今回あえて入れないもの: キャラ化、ネタ枠、季節装飾、感情芝居、長文囲み専用パーツ

## 3. set 構成
- 最上段に置く高頻度絵文字: `S01 重要`, `S02 確認`, `S03 お願い`, `S04 共有`
- 必須絵文字: 重要、確認、お願い、共有、注目ペア1組、注意、補足、要返信、了解、決定、あとで、締切、メモ、おすすめ、結論
- 差分の軸: 機能、向き、中央サイン、外形の切れ方、開閉方向
- 主力枠と補助枠の配分: 主力枠 `14`、組み合わせ補助枠 `2`
- 組み合わせ補助枠の扱い: `S05 みて左` と `S06 みて右` の2枠のみ。囲み補助は短い語に限定し、set の多数派にはしない。
- 重複禁止ルール: 左右差分は `S05 / S06` のみ許容。`重要 / 注意 / 締切` は色ではなく外形と中央サインで分ける。`確認 / 要返信 / 了解` は役割の向きで差を立てる。
- 並び順方針: 上段は高頻度の独立用途、2段目に導線補助と注意・補足、3段目に依頼応答と状態、4段目に締めや軽い日常用途を置く。
- セット全体の温度感: 実用寄りだが冷たすぎない。事務用品の整理感に、日常会話でも使えるやわらかさを足す。

## 4. 個別絵文字の設計メモ
- `S01 / 重要`
  - 用途: 最重要の一言を立てる
  - 単体送信時の意味: `大事`
  - 文中使用時の役割: 重要語の前置き
  - 連続使用メモ: 原則1個で立てる
  - rough で先に確認したい点: `注意` と差が十分か
- `S02 / 確認`
  - 用途: 確認依頼、見直し
  - 単体送信時の意味: `確認して`
  - 文中使用時の役割: 要確認項目の頭
  - 連続使用メモ: 箇条書き風にも使える
  - rough で先に確認したい点: `要返信` や `了解` と混ざらないか
- `S03 / お願い`
  - 用途: 依頼、頼みごと
  - 単体送信時の意味: `お願いします`
  - 文中使用時の役割: 依頼文の頭
  - 連続使用メモ: `確認` と並んでも役割差が見えること
  - rough で先に確認したい点: 圧が出すぎないか
- `S04 / 共有`
  - 用途: 情報共有、お知らせ
  - 単体送信時の意味: `共有です`
  - 文中使用時の役割: 周知文の頭
  - 連続使用メモ: `重要` と並ぶ場合も意味差が見えること
  - rough で先に確認したい点: `メモ` と混ざらないか
- `S05 / みて左`
  - 用途: 重要語の左側補助、見出し開始
  - 単体送信時の意味: `見て / ここ`
  - 文中使用時の役割: テキストの左に置いて注目を集める
  - 連続使用メモ: `S06` と対で短い語を囲める
  - rough で先に確認したい点: 単体で未完成に見えないか
- `S06 / みて右`
  - 用途: 重要語の右側補助、見出し終わり
  - 単体送信時の意味: `ここ / 注目`
  - 文中使用時の役割: テキストの右に置いて注目の締めを作る
  - 連続使用メモ: `S05` と対で使用
  - rough で先に確認したい点: `S05` と並べたときだけでなく、単独でも読めるか
- `S07 / 注意`
  - 用途: 注意点、禁止寄りではない警告
  - 単体送信時の意味: `注意`
  - 文中使用時の役割: リスクや補足の手前
  - 連続使用メモ: `重要` と並べると意味が近いため原則単独
  - rough で先に確認したい点: 危険標識に寄りすぎないか
- `S08 / 補足`
  - 用途: 追記、備考
  - 単体送信時の意味: `補足あり`
  - 文中使用時の役割: 本文後の追記ラベル
  - 連続使用メモ: 1行に1個まで
  - rough で先に確認したい点: `結論` と混ざらないか
- `S09 / 要返信`
  - 用途: 返信依頼、返答待ち
  - 単体送信時の意味: `返信ください`
  - 文中使用時の役割: 返答が必要な一文の頭
  - 連続使用メモ: `お願い` と並べても役割差が見えること
  - rough で先に確認したい点: `確認` と混ざらないか
- `S10 / 了解`
  - 用途: 受領、了承
  - 単体送信時の意味: `了解`
  - 文中使用時の役割: 了承や受領の短い返答
  - 連続使用メモ: 1個で完結
  - rough で先に確認したい点: `確認` や `要返信` と方向差が読めるか
- `S11 / 決定`
  - 用途: 確定、決まり
  - 単体送信時の意味: `決定`
  - 文中使用時の役割: 決定事項の頭
  - 連続使用メモ: `結論` と併用する場合も役割差が見えること
  - rough で先に確認したい点: `結論` と違いが立つか
- `S12 / あとで`
  - 用途: 保留、後回し
  - 単体送信時の意味: `あとで`
  - 文中使用時の役割: 今は処理しない項目の頭
  - 連続使用メモ: `メモ` と近づきすぎないこと
  - rough で先に確認したい点: 保留と放置の違いが読めるか
- `S13 / 締切`
  - 用途: 期限、急ぎ、締切共有
  - 単体送信時の意味: `急ぎ / 期限`
  - 文中使用時の役割: 日付や時間の前置き
  - 連続使用メモ: `重要` と併記する場合は意味重複に注意
  - rough で先に確認したい点: 急ぎ感を出しつつうるさくなりすぎないか
- `S14 / メモ`
  - 用途: メモ、控え、記録
  - 単体送信時の意味: `メモ`
  - 文中使用時の役割: メモ書きの頭
  - 連続使用メモ: `共有` や `補足` と差が必要
  - rough で先に確認したい点: `共有` と混ざらないか
- `S15 / おすすめ`
  - 用途: 推奨、勧めたいこと
  - 単体送信時の意味: `おすすめ`
  - 文中使用時の役割: 推したい情報の頭
  - 連続使用メモ: 1個で立てる
  - rough で先に確認したい点: 実用品 set の中で浮きすぎないか
- `S16 / 結論`
  - 用途: まとめ、最終判断
  - 単体送信時の意味: `これが結論`
  - 文中使用時の役割: 最終文の頭
  - 連続使用メモ: 1個で完結させる
  - rough で先に確認したい点: `補足` と視覚的に逆方向の意味が出るか

## 7. quality gate メモ
- rough 前に止める問題: 1機能1外形が曖昧、左右ペア以外の重複、色頼み差分、主力14枠の意味が薄い
- handoff 前に止める問題: 各絵文字の単体意味が説明できない、`補足 / 結論` の差が弱い、`確認 / 要返信 / 了解` の方向差が弱い
- 提出前に止める問題: 小表示で中央サインが潰れる、タブ画像がブランド核を示せない、metadata と用途のズレ
- `Watch` として継続監視する問題: 左右ペアの使用頻度、仕事向けに寄りすぎて日常接続性が落ちないか、`おすすめ` が set 内で浮きすぎないか
