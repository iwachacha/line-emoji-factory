# release-001 セット仕様

この release は、`ゆるゲージ` の最初のコア商品である。
「見た / 了解 / 対応中 / 完了」を最上段に置き、短い返答と進捗状態を16個で販売可能な密度にする。

## 1. 基本情報
- ブランド名: `ゆるゲージ`
- release / set 名: `進捗と返事コア16`
- 参照するブランド設定: `brand/brand-setting.md`
- 参照する制作基盤: `brand/brand-production-brief.md`
- 判定状態: `Design Ready`
- 商品種別: `静止絵文字`
- 初期セット個数: `16`
- 初期セット個数の根拠: 高頻度返答4、確認・共有4、保留・期限4、調整・締め4で、初回商品としての使用幅と差分明確性が両立する。
- この release の一言コンセプト: 返事と進み具合を、1文字ゲージで軽く返す。
- ブランド内での役割: ブランドの視覚文法と基本語彙を固定する初回コア。
- 今回の市場 / 会話観測サマリー: 敬語・仕事絵文字は多いが、進み具合をゲージ文法で統一する状態共有は薄い。アニメーションやIP人気は追わず、静止で即読できる実用品として差別化する。

## 2. release の目的
- この release で取りにいく使用場面: 確認済み返信、進行報告、完了報告、保留、急ぎ、締切、共有、感謝、未定・決定の整理。
- 主な相手 / 関係性: 同僚、家族、グループ運営、友人との予定調整、個人メモ。
- 先に解くべき会話課題: 文章にすると重く、既存記号だと冷たい状態返答を、低圧で可視化する。
- 優先検証シーン: 単体返信、文末補助、2個連続の状態推移、仕事寄り、日常寄り、少し急ぎの文脈。
- 今回あえて入れないもの: 長い敬語文、季節、キャラ表情、左右ペア、装飾目的だけの記号。

## 3. set 構成
- 最上段に置く高頻度絵文字: `001 見`, `002 了`, `003 中`, `004 済`
- 必須絵文字: 見た、了解、対応中、完了、確認、要返信、共有、ありがとう、保留、あとで、急ぎ、締切、調整、未定、決定、休憩。
- 差分の軸: 中央ラベル、バー量、補助サイン、アクセント色、用途カテゴリ。
- 主力枠と補助枠の配分: 主力12、補助4。
- 組み合わせ補助枠の扱い: なし。2個連続は状態推移の任意用途に留める。
- 重複禁止ルール: `了 / 済 / 決` は「了承 / 完了 / 決定」、`中 / 調` は「作業中 / 調整中」、`見 / 確 / 返` は「見た / 確認依頼 / 返信依頼」として役割を分ける。
- 並び順方針: 上段は最頻出返答、2段目は依頼と共有、3段目は保留と期限、4段目は調整と締め。
- セット全体の温度感: 低圧、実用、少し丸い。仕事にも日常にも置ける。

## 4. 個別絵文字の設計メモ
- `001 / 見`
  - 用途: 既読だけでなく、確認したことを軽く伝える
  - 単体送信時の意味: `見ました`
  - 文中使用時の役割: 確認済みの短い印
  - 連続使用メモ: `返` と並べると「見たので返信待ち」
  - rough で先に確認したい点: `確` と依頼方向が混ざらないか
- `002 / 了`
  - 用途: 了解、了承
  - 単体送信時の意味: `了解`
  - 文中使用時の役割: 返答の締め
  - 連続使用メモ: 原則単独
  - rough で先に確認したい点: `済` と完了意味が混ざらないか
- `003 / 中`
  - 用途: 対応中、進行中
  - 単体送信時の意味: `対応中`
  - 文中使用時の役割: 作業が進行している印
  - 連続使用メモ: `済` と並べると進捗推移
  - rough で先に確認したい点: `調` と混ざらないか
- `004 / 済`
  - 用途: 完了、処理済み
  - 単体送信時の意味: `完了`
  - 文中使用時の役割: 終わった項目の印
  - 連続使用メモ: `中 → 済` で状態推移
  - rough で先に確認したい点: `決` と混ざらないか
- `005 / 確`
  - 用途: 確認依頼
  - 単体送信時の意味: `確認して`
  - 文中使用時の役割: 確認してほしい一文の頭
  - 連続使用メモ: `返` と並べると依頼が強くなりすぎるため注意
  - rough で先に確認したい点: 依頼方向が読めるか
- `006 / 返`
  - 用途: 要返信、返答待ち
  - 単体送信時の意味: `返信ください`
  - 文中使用時の役割: 返事が必要な項目の印
  - 連続使用メモ: 原則1個
  - rough で先に確認したい点: 圧が強すぎないか
- `007 / 共`
  - 用途: 共有、お知らせ
  - 単体送信時の意味: `共有です`
  - 文中使用時の役割: 情報共有の頭
  - 連続使用メモ: `見` と並べても意味差が残ること
  - rough で先に確認したい点: `メモ` 系に見えないか
- `008 / 礼`
  - 用途: ありがとう、助かった
  - 単体送信時の意味: `ありがとう`
  - 文中使用時の役割: 感謝を軽く添える
  - 連続使用メモ: 1個で完結
  - rough で先に確認したい点: 実用セット内で浮きすぎないか
- `009 / 待`
  - 用途: 保留、待ち
  - 単体送信時の意味: `保留`
  - 文中使用時の役割: 今は止める項目の印
  - 連続使用メモ: `後` と併用しない
  - rough で先に確認したい点: 待機と未定の差が読めるか
- `010 / 後`
  - 用途: あとで対応
  - 単体送信時の意味: `あとで`
  - 文中使用時の役割: 後回しの印
  - 連続使用メモ: `中` と並べると曖昧になるため避ける
  - rough で先に確認したい点: 放置ではなく後でやると読めるか
- `011 / 急`
  - 用途: 急ぎ、優先度高め
  - 単体送信時の意味: `急ぎ`
  - 文中使用時の役割: 優先度の印
  - 連続使用メモ: 連打禁止
  - rough で先に確認したい点: うるさくなりすぎないか
- `012 / 締`
  - 用途: 締切、期限
  - 単体送信時の意味: `締切`
  - 文中使用時の役割: 日付や時間の前置き
  - 連続使用メモ: `急` と並べると圧が強いため原則単独
  - rough で先に確認したい点: `急` と用途差が見えるか
- `013 / 調`
  - 用途: 調整中、相談中
  - 単体送信時の意味: `調整中`
  - 文中使用時の役割: 未確定だが動いている印

  - 単体送信時の意味: `未定`
  - 文中使用時の役割: 決定前の印
  - 連続使用メモ: `決` と対で使える
  - rough で先に確認したい点: ネガティブに寄りすぎないか
- `015 / 決`
  - 用途: 決定、確定
  - 単体送信時の意味: `決定`
  - 文中使用時の役割: 確定事項の印
  - 連続使用メモ: `未 → 決` で推移
  - rough で先に確認したい点: `済` と区別できるか
- `016 / 休`
  - 用途: 休憩、おつかれ、いったん休み
  - 単体送信時の意味: `休憩`
  - 文中使用時の役割: 作業や会話を一拍緩める
  - 連続使用メモ: 1個で完結
  - rough で先に確認したい点: 商品コンセプトから浮かないか

## 7. quality gate メモ
- rough 前に止める問題: ゲージ文法が説明できない、1文字ラベルが多義すぎる、色違いだけの差分。
- handoff 前に止める問題: `了 / 済 / 決`、`中 / 調`、`見 / 確 / 返` の役割差が弱い。
- 提出前に止める問題: 180pxで可読性が弱い、64px相当で主要意味が読めない、PNG透過や容量仕様を満たさない。
- `Watch` として継続監視する問題: 仕事寄りすぎ、1文字ラベルの文化差、ゲージ量が意味として伝わるか。
- 優先監査対象の絵文字: `002`, `003`, `004`, `005`, `006`, `013`, `015`
